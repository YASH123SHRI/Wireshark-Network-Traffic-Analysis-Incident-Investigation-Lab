# ARP Traffic Analysis

## Objective

Analyze Address Resolution Protocol (ARP) packets captured using Wireshark and understand how devices resolve IP addresses to MAC addresses within a local network.

---

## Background

ARP (Address Resolution Protocol) is used in IPv4 networks to map an IP address to a physical MAC address before communication begins on a local network.

---

## Traffic Generation

ARP traffic was generated by clearing the ARP cache and sending an ICMP request to the default gateway.

Commands:

```cmd
arp -d *
ping <Default_Gateway_IP> -n 3
```

Example:

```cmd
arp -d *
ping 192.168.1.1 -n 3
```

---

## Packet Capture

1. Open Wireshark.
2. Start capturing on the active network interface.
3. Execute the commands above.
4. Stop the capture.
5. Save the file as:

```
arp.pcapng
```

---

## Display Filter

```
arp
```

---

## Packet Analysis

The capture contains:

- ARP Request
- ARP Reply
- Source MAC Address
- Destination MAC Address
- Sender IP Address
- Target IP Address

---

## Important Packet Fields

| Field | Description |
|-------|-------------|
| Opcode | Request (1) / Reply (2) |
| Sender MAC | MAC address of requesting host |
| Sender IP | IP address of requesting host |
| Target MAC | MAC address of destination |
| Target IP | Requested IP address |

---

## Findings

- ARP Request broadcasted to the local network.
- Router responded with its MAC address.
- Address resolution completed successfully.

---

## Conclusion

ARP packets demonstrated the process of resolving an IPv4 address into a MAC address before communication begins on the local network.
