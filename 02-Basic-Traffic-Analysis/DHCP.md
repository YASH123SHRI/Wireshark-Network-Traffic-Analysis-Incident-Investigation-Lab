# DHCP Traffic Analysis

## Objective

Capture and analyze Dynamic Host Configuration Protocol (DHCP) traffic to understand how a client obtains an IP address automatically from a DHCP server.

---

## Background

DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses and network configuration parameters to devices joining a network. The standard DHCP process follows four steps: Discover, Offer, Request, and Acknowledgment (DORA).

---

## Traffic Generation

DHCP traffic was generated by releasing and renewing the network configuration.

Commands:

```cmd
ipconfig /release
ipconfig /renew
```

Alternatively, disconnecting and reconnecting the network adapter also generates DHCP traffic.

---

## Packet Capture

1. Open Wireshark.
2. Select the active network interface.
3. Start packet capture.
4. Execute the DHCP commands.
5. Stop the capture.
6. Save the file as:

```
dhcp.pcapng
```

---

## Display Filter

```
dhcp
```

or

```
bootp
```

---

## Packet Analysis

The capture contains the DHCP DORA process:

- DHCP Discover
- DHCP Offer
- DHCP Request
- DHCP Acknowledgment (ACK)

---

## Important Packet Fields

| Field | Description |
|--------|-------------|
| Transaction ID | Identifies the DHCP transaction |
| Client MAC Address | Hardware address of the requesting client |
| Offered IP Address | IP address proposed by the DHCP server |
| DHCP Message Type | Discover, Offer, Request, ACK |
| Lease Time | Duration of the assigned IP address |

---

## Findings

- Client successfully requested an IP address.
- DHCP server responded with an available address.
- The client accepted the offered configuration.
- Network configuration was completed successfully.

---

## Conclusion

DHCP packet analysis demonstrated the complete IP address assignment process using the DORA sequence. This protocol enables automatic network configuration for client devices.
