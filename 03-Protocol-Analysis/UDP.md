# UDP Traffic Analysis

## Objective

Capture and analyze User Datagram Protocol (UDP) traffic using Wireshark.

---

## Background

UDP (User Datagram Protocol) is a connectionless transport-layer protocol. Unlike TCP, UDP does not establish a connection before sending data and does not guarantee packet delivery.

---

## Traffic Generation

UDP traffic was generated by performing a DNS lookup.

Command

```cmd
nslookup openai.com
```

---

## Packet Capture

1. Start Wireshark.
2. Begin packet capture.
3. Execute the command above.
4. Stop capture.
5. Save as:

```
udp.pcapng
```

---

## Display Filter

```
udp
```

---

## Packet Analysis

Observed:

- UDP Source Port
- UDP Destination Port
- Length
- Checksum

---

## Important Packet Fields

| Field | Description |
|--------|-------------|
| Source Port | Client port |
| Destination Port | Service port |
| Length | UDP packet size |
| Checksum | Error detection |

---

## Findings

- UDP transmitted packets without establishing a connection.
- No acknowledgment mechanism was present.
- DNS communication used UDP port 53.

---

## Conclusion

UDP provides fast, lightweight communication suitable for applications where speed is more important than guaranteed delivery.
