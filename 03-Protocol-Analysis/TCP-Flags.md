# TCP Flags Analysis

## Objective

Analyze TCP control flags using Wireshark to understand how TCP manages connection establishment, data transfer, and connection termination.

---

## Background

TCP uses control flags to manage communication between hosts. These flags indicate the state and purpose of each TCP packet.

Common TCP flags include:

- SYN
- ACK
- FIN
- RST
- PSH
- URG

---

## Traffic Generation

TCP traffic was generated by visiting a website and establishing a TCP connection.

Example:

```
https://example.com
```

---

## Packet Capture

1. Open Wireshark.
2. Select the active network interface.
3. Start packet capture.
4. Visit a website.
5. Close the browser tab.
6. Stop the capture.
7. Save the capture as:

```
tcp-flags.pcapng
```

---

## Display Filters

Show all TCP packets:

```
tcp
```

SYN packets:

```
tcp.flags.syn == 1
```

FIN packets:

```
tcp.flags.fin == 1
```

RST packets:

```
tcp.flags.reset == 1
```

---

## Packet Analysis

The capture contains several TCP control flags used during communication.

| Flag | Purpose |
|------|---------|
| SYN | Starts a TCP connection |
| ACK | Acknowledges received packets |
| FIN | Gracefully closes a connection |
| RST | Abruptly resets a connection |
| PSH | Delivers data immediately |
| URG | Marks urgent data |

---

## Important Packet Fields

| Field | Description |
|--------|-------------|
| Source Port | Client Port |
| Destination Port | Server Port |
| Sequence Number | Packet sequence |
| Acknowledgment Number | Confirms received data |
| TCP Flags | Communication state |

---

## Findings

- SYN initiated the connection.
- ACK acknowledged successful communication.
- FIN closed the session gracefully.
- PSH packets carried application data.

---

## Conclusion

TCP flags coordinate every stage of a TCP connection, ensuring reliable communication and proper session management.
