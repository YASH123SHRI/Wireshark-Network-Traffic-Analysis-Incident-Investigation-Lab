# Network Conversations Analysis

## Objective

Analyze network conversations using Wireshark to identify communication between endpoints.

---

## Background

A conversation represents all packets exchanged between two network endpoints. Wireshark groups these communications to simplify traffic analysis.

---

## Traffic Generation

Network traffic was generated by:

- Browsing websites
- Running ping
- Performing DNS lookups

---

## Packet Capture

1. Start Wireshark.
2. Capture normal network activity.
3. Stop capture.
4. Save as:

```
conversations.pcapng
```

---

## Conversation Analysis

Navigate to:

```
Statistics
    ↓
Conversations
```

Review the following tabs:

- Ethernet
- IPv4
- IPv6
- TCP
- UDP

---

## Information Observed

Each conversation includes:

- Source Address
- Destination Address
- Packets
- Bytes
- Duration

---

## Findings

- Multiple TCP conversations were identified.
- DNS queries generated UDP conversations.
- HTTPS sessions generated TCP conversations.
- ICMP traffic appeared as separate conversations.

---

## Conclusion

Conversation statistics provide a high-level overview of network communication and help identify active hosts, protocols, and data transfer volumes.
