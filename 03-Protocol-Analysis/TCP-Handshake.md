# TCP Three-Way Handshake Analysis

## Objective

Capture and analyze the TCP Three-Way Handshake to understand how a reliable TCP connection is established between a client and a server.

---

## Background

TCP (Transmission Control Protocol) establishes a reliable connection before transmitting data. This process is known as the Three-Way Handshake and consists of three packets:

1. SYN
2. SYN, ACK
3. ACK

This ensures that both the client and the server are ready to exchange data.

---

## Traffic Generation

A TCP connection was generated by visiting a secure website.

Example:

```
https://example.com
```

---

## Packet Capture

1. Open Wireshark.
2. Select the active network interface.
3. Start packet capture.
4. Visit `https://example.com` in a web browser.
5. Stop packet capture.
6. Save the capture as:

```
tcp-handshake.pcapng
```

---

## Display Filter

```
tcp.flags.syn==1
```

or

```
tcp.stream == 0
```

---

## Packet Analysis

The TCP connection establishment consists of:

| Step | Description |
|------|-------------|
| SYN | Client requests a connection |
| SYN, ACK | Server acknowledges the request |
| ACK | Client confirms the connection |

---

## Important Packet Fields

| Field | Description |
|--------|-------------|
| Source Port | Client port |
| Destination Port | Server port |
| Sequence Number | Initial sequence number |
| Acknowledgment Number | Confirms received packets |
| Window Size | Available receive buffer |
| Flags | SYN, ACK |

---

## Findings

- TCP connection established successfully.
- Three-way handshake completed without errors.
- Sequence and acknowledgment numbers incremented correctly.

---

## Conclusion

The TCP Three-Way Handshake ensures reliable communication before data transmission begins, forming the foundation of all TCP-based applications.
